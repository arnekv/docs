The tutorial introduction should include the following in a short paragraph:

- Clarify **audience**.
- State **prerequisites** and prior knowledge needed.
- State **what** the user will accomplish or build and the **user problem** it solves.
- Link to an **example** of the project the user will complete.

## What you will do

To do x, follow these steps:

<Steps>
  <Step title="Action the user will take">
    1. In one sentence, describe what the user will do in this step.
    2. If necessary, break down the tasks into chunks, allowing the user to scan quickly to find their place if they navigated away from this screen to perform the task.

        Avoid replicating conceptual information that is covered elsewhere, provide inline links instead. Only include conceptual information unique to this use case.

    1. For UI based tasks, include the button or options the users should click.
    1. If the task adds code, include the code in context.
  </Step>

  <Step title="Do the next thing">
    1. Rinse and repeat, adding steps and tasks until the tutorial is complete.
  </Step>

</Steps>

## Further reading

Include a bulleted list of tutorials or articles the user can reference to extend the concepts taught in this tutorial.
