<Warning>
This guide covers breaking changes and migration steps for major versions of the Cognite JavaScript SDK. Review carefully before upgrading.
</Warning>

## Upgrade from 2.x to 3.x

<Info>
Version 3.x introduces several breaking changes to improve consistency and developer experience. Update your code according to the sections below.
</Info>

### Renamed interfaces

Some TypeScript interfaces have been renamed for consistency between resource types. Common changes:

- `GetTimeSeriesMetadataDTO` → `Timeseries`
- `PostTimeSeriesMetadataDTO` → `ExternalTimeseries`
- `DatapointsGetAggregateDatapoint` → `DatapointAggregates`
- `FilesMetadata` → `FileInfo`

See the [changelog](https://github.com/cognitedata/cognite-sdk-js/blob/master/packages/stable/CHANGELOG.md#300-2020-08-03) for a full list.

### Events aggregate endpoints moved

- `client.events.aggregate(...)` → `client.events.aggregate.count()`
- `client.events.uniqueValuesAggregate(...)` → `client.events.aggregate.uniqueValues(...)`

### Timestamp-to-Date auto-conversion

<Warning>
The SDK now automatically converts response timestamp values to date objects. This may introduce breaking changes, especially if you use the Raw API. See [issue #333](https://github.com/cognitedata/cognite-sdk-js/issues/333).
</Warning>

### Helper classes removed

The following helper classes have been removed: `AssetClass`, `TimeSeriesClass`, `TimeSeriesList`, and `AssetList`.

<Tip>
You can re-implement most helper methods using other SDK calls. See examples below.
</Tip>

**Asset.delete()**

```js title="Re-implement Asset.delete()"
public async delete(options: DeleteOptions = {}) {
  return this.client.assets.delete(
    [{ id: this.id }],
    options
  );
}
```

**Asset.subtree()**

```js title="Re-implement Asset.subtree()"
public async subtree() {
  const { items } = await client.assets.list({
    filter: {
      assetSubtreeIds: [{ id: this.id }],
    }
  });
}
```

**TimeSeriesList.getAllDatapoints(...)**

```js title="Re-implement getAllDatapoints()"
public getAllDatapoints = async (options: DatapointsMultiQueryBase = {}) => {
  const tsIds = this.map(({ id }) => ({ id }));
  return this.client.datapoints.retrieve({
    items: tsIds,
    ...options,
  });
};
```

### Removed `client.assets.retrieveSubtree(...)`

Use the new `Asset.subtree()` implementation above.

### Change filter passing to `client.timeseries.list`

The function now takes a `TimeseriesFilterQuery` (matching the [API spec](/api#operation/listTimeSeries)). All filter fields should be inside the `filter` object.

```js title="Old way"
// Old way
const timeseries = await client.timeseries.list({
  assetIds: [1, 2],
  unit: 'liter',
  limit: 100,
});
```

```js title="New way"
// New way
const timeseries = await client.timeseries.list({
  filter: { assetIds: [1, 2], unit: 'liter' },
  limit: 100,
});
```

## Upgrade from 1.x to 2.x

<Info>
Version 2.x introduced a new client-based API and authentication flow. Update your code as shown below.
</Info>

### Constructor and authentication

You must create a new SDK instance before using the SDK:

```js title="Create client instance"
const client = new CogniteClient({ appId: 'YOUR APPLICATION NAME' });
```

Authenticate your client:

<Tabs>
<Tab title="API Key">

```js title="API Key authentication"
client.loginWithApiKey({ project, apiKey });
```

</Tab>
<Tab title="OAuth">

```js title="OAuth authentication"
client.loginWithOAuth({
  project: 'publicdata',
});
```

</Tab>
</Tabs>

For more info about OAuth authentication, see the [authentication guide](https://github.com/cognitedata/cognite-sdk-js/blob/master/guides/authentication.md).
